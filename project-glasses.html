3D Audio & AI-Powered Obstacle Recognition Glasses

ongoing

Smart glasses for visually impaired users — combining real-time YOLO object detection on a Raspberry Pi with spatial 3D audio cues that guide navigation without requiring sight.

Motivation

Over 2.2 billion people worldwide have some form of vision impairment. Existing assistive technologies — white canes and guide dogs — provide limited spatial information and no object identification. This project aims to build a wearable, affordable alternative that translates visual scene understanding into intuitive spatial audio, allowing users to hear where obstacles and objects are located in 3D space around them without any visual input.

Project Goals

  • Detect and classify objects in real time using a YOLO-based model running on edge hardware.
  • Convey object identity and position through directional 3D audio using Head-Related Transfer Functions (HRTF).
  • Design lightweight, wearable hardware with dual cameras for depth estimation and wide field-of-view coverage.
  • Achieve low-latency inference (<200 ms) suitable for safe real-world navigation.
  • Develop a user-centered interaction model minimising cognitive load.

System Architecture

Perception — Dual Camera

Two cameras mounted on the glasses frame provide stereo vision for rough depth estimation. Each frame is streamed to the Raspberry Pi for real-time inference.

Detection — YOLOv8 (Edge)

A quantised YOLOv8-nano model performs object detection at the highest frame rate achievable on the Pi's ARM core. Detected bounding boxes yield object class and screen-space position.

Depth Estimation — CNN/RNN

A lightweight convolutional-recurrent model maps the stereo image pair to a per-object distance estimate, feeding into the audio spatialisation step.

Spatialisation — 3D Audio (HRTF)

Object class and estimated azimuth/elevation are encoded as 3D binaural audio using HRTF filters. Each object class is mapped to a distinct earcon; proximity modulates volume.

Output — Bone-Conduction Headphones

Bone-conduction headphones allow users to hear spatial audio feedback while remaining aware of ambient environmental sounds — a key safety requirement.

Processing Unit — Raspberry Pi 5

Runs the full inference stack: camera capture, YOLO detection, depth estimation, and real-time HRTF audio rendering in a closed loop.

Real-Time Pipeline

  1. Frame capture — Both cameras stream frames; one is selected for detection, the pair is used for depth.
  2. Object detection — YOLOv8-nano runs inference; outputs class labels, confidence scores, and bounding box coordinates.
  3. Distance estimation — Stereo disparity map or CNN model estimates metric distance to each detected object.
  4. Scene prioritisation — Objects are ranked by proximity and danger level; only the most relevant 1–3 objects are sonified per cycle to avoid information overload.
  5. 3D audio rendering — HRTF filters spatialise a short earcon for each object at its estimated azimuth and elevation; rendered to stereo and played through bone-conduction headphones.

Current Status

  • YOLOv8-nano benchmarked on Raspberry Pi 5; real-time inference at ~8–12 FPS confirmed.
  • HRTF-based 3D audio spatialisation prototype completed in Python using the sounddevice and scipy libraries.
  • Dual-camera hardware design in progress; depth estimation module under development.
  • User interaction model and cognitive-load evaluation planned for next phase.

Upcoming Work

  • Integrate stereo depth estimation with the detection pipeline end-to-end.
  • Fine-tune YOLO on indoor obstacle classes (chairs, doors, stairs, people).
  • Conduct user trials with visually impaired participants to evaluate spatial audio clarity and navigation accuracy.
  • Optimise pipeline latency to sub-200 ms using model quantisation and thread parallelism.
  • Design and 3D-print the final wearable glasses frame with integrated cameras and battery pack.
